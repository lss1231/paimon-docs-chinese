    Quick Start #

    本指南用于指导在 Flink 中使用 Paimon。

    Jar包 Jars #

    Paimon 目前支持 Flink 2.0、1.20、1.19、1.18、1.17、1.16、1.15。我们推荐使用最新的 Flink 版本以获得更好的体验。

    请下载对应版本的 jar 文件。

    当前，Paimon 提供两种类型的 jar 文件：一种（bundled jar）用于读写数据，另一种（action jar）用于手动 compaction 等操作，

    Version Type Jar
    Flink 2.0 Bundled Jar paimon-flink-2.0-1.2.0.jar
    Flink 1.20 Bundled Jar paimon-flink-1.20-1.2.0.jar
    Flink 1.19 Bundled Jar paimon-flink-1.19-1.2.0.jar
    Flink 1.18 Bundled Jar paimon-flink-1.18-1.2.0.jar
    Flink 1.17 Bundled Jar paimon-flink-1.17-1.2.0.jar
    Flink 1.16 Bundled Jar paimon-flink-1.16-1.2.0.jar
    Flink 1.15 Bundled Jar paimon-flink-1.15-1.2.0.jar
    Flink Action Action Jar paimon-flink-action-1.2.0.jar

    你也可以从源码手动构建 bundled jar。

    要从源码构建，请先 克隆 git 仓库。

    使用以下命令构建 bundled jar。

    • mvn clean install -DskipTests

    你可以在 ./paimon-flink/paimon-flink-<flink-version>/target/paimon-flink-<flink-version>-1.2.0.jar 找到 bundled jar，在 ./paimon-flink/paimon-flink-action/target/paimon-flink-action-1.2.0.jar 找到 action jar。

    开始 Start #

    Step 1: 下载 Flink

    如果你还没有下载Flink，你可以download Flink，然后使用以下命令解压归档文件：

    tar -xzf flink-*.tgz

    Step 2: 拷贝 Paimon Bundled Jar

    拷贝paimon的bundled jar报到 Flink 安装路径下的 lib 文件夹。

    cp paimon-flink-*.jar <FLINK_HOME>/lib/

    Step 3: 拷贝 Hadoop Bundled Jar

    如果机器处于 Hadoop 环境中，请确保环境变量 HADOOP_CLASSPATH 的值包含指向 Hadoop 公共库的路径，无需使用以下预打包的 Hadoop jar。

    下载 预打包的 Hadoop jar，并将该 jar 文件复制到 Flink 安装目录下的 lib 目录。

    cp flink-shaded-hadoop-2-uber-*.jar <FLINK_HOME>/lib/

    Step 4: 启动一个Flink本地集群Local Cluster

    为了同时运行多个 Flink 作业，你需要修改 <FLINK_HOME>/conf/flink-conf.yaml（Flink 版本 < 1.19）或 <FLINK_HOME>/conf/config.yaml（Flink 版本 >= 1.19）中的集群配置。

    taskmanager.numberOfTaskSlots: 2

    要启动本地集群，运行 Flink 自带的 bash 脚本：

    <FLINK_HOME>/bin/start-cluster.sh

    你可以通过浏览器访问 localhost:8081 来查看 Flink 的 Web UI，确认集群已经启动并运行。

    此时，你可以启动 Flink SQL 客户端来执行 SQL 脚本。

    <FLINK_HOME>/bin/sql-client.sh

    Step 5: 创建一个Catalog和一张表

    -- 如果你在分布式环境中尝试 Paimon，
-- warehouse 路径应该设置为共享文件系统，例如 HDFS 或 OSS
CREATE CATALOG my_catalog WITH (
    'type'='paimon',
    'warehouse'='file:/tmp/paimon'
);

USE CATALOG my_catalog;

-- 创建一张 word count 表
CREATE TABLE word_count (
    word STRING PRIMARY KEY NOT ENFORCED,
    cnt BIGINT
);

    使用 FlinkGenericCatalog 时，需要使用 Hive metastore。这样，你就可以同时使用 Paimon、Hive 以及 Flink Generic 表（Kafka 等）中的所有表！

    在这种模式下，创建表时应使用 connector 选项。

    Paimon 会使用你 hive-site.xml 中的 hive.metastore.warehouse.dir，请确保使用带 scheme 的路径， 例如 hdfs://...。否则，Paimon 将使用本地路径。 
    CREATE CATALOG my_catalog WITH (
    'type'='paimon-generic',
    'hive-conf-dir'='...',
    'hadoop-conf-dir'='...'
);

USE CATALOG my_catalog;

-- 创建一张 word count 表
CREATE TABLE word_count (
    word STRING PRIMARY KEY NOT ENFORCED,
    cnt BIGINT
) WITH (
    'connector'='paimon'
);

    Step 6: 写入数据

    -- create a word data generator table
CREATE TEMPORARY TABLE word_table (
    word STRING
) WITH (
    'connector' = 'datagen',
    'fields.word.length' = '1'
);

-- paimon requires checkpoint interval in streaming mode
SET 'execution.checkpointing.interval' = '10 s';

-- write streaming data to dynamic table
INSERT INTO word_count SELECT word, COUNT(*) FROM word_table GROUP BY word;

    Step 7: OLAP 查询

    -- use tableau result mode
SET 'sql-client.execution.result-mode' = 'tableau';

-- switch to batch mode
RESET 'execution.checkpointing.interval';
SET 'execution.runtime-mode' = 'batch';

-- olap query the table
SELECT * FROM word_count;

    你可以多次执行查询，并观察结果的变化。

    Step 8: 流式查询

    -- switch to streaming mode
SET 'execution.runtime-mode' = 'streaming';

-- track the changes of table and calculate the count interval statistics
SELECT `interval`, COUNT(*) AS interval_cnt FROM
    (SELECT cnt / 10000 AS `interval` FROM word_count) GROUP BY `interval`;

    Step 9: 退出

    在 localhost:8081 取消 streaming 作业，然后执行以下 SQL 脚本退出 Flink SQL 客户端。

    -- uncomment the following line if you want to drop the dynamic table and clear the files
-- DROP TABLE word_count;

-- exit sql-client
EXIT;

    停止Flink 本地集群

    ./bin/stop-cluster.sh

    Use Flink Managed Memory #

    Paimon 任务可以基于 executor 内存创建内存池，这些内存池将由 Flink executor 管理，例如 Flink Task Manager 中的 managed memory。通过 executor 管理多个任务的 writer buffer，可以提升 sink 的稳定性和性能。

    如果使用 Flink managed memory，可以设置以下属性：

    选项 Option 默认值 Default 描述 Description
    sink.use-managed-memory-allocator false 如果设置为 true，Flink sink 将使用管理内存（managed memory）用于 merge tree；否则，将创建独立的内存分配器，这意味着每个任务都会分配并管理自己的内存池（堆内存）。如果在一个 Executor 中任务过多，可能会导致性能问题甚至 OOM。
    sink.managed.writer-buffer-memory 256M 管理内存中 writer 缓冲区的权重，Flink 会根据该权重计算 writer 的内存大小，实际使用的内存取决于运行环境。目前，该属性定义的内存大小等于运行时分配给写缓冲区的实际内存。

    Use In SQL 用户可以在 SQL 中为 Flink 管理内存设置内存权重，然后 Flink sink 操作符将获取内存池大小并为 Paimon writer 创建分配器。

    INSERT INTO paimon_table /*+ OPTIONS('sink.use-managed-memory-allocator'='true', 'sink.managed.writer-buffer-memory'='256M') */
SELECT * FROM ....;

    设置动态选项 Setting dynamic options #

    在与 Paimon 表交互时，可以在不修改 catalog 配置的情况下调整表级选项。Paimon 会提取作业级的动态配置，并在当前会话生效。

    • 动态表选项的 key 格式为：paimon.${catalogName}.${dbName}.${tableName}.${config_key}。其中 catalogName/dbName/tableName 可以使用 *，表示匹配所有对应部分。
    • 动态全局选项的 key 格式为：${config_key}，全局选项会对所有表生效。如果表选项与全局选项冲突，表选项优先。

    例如：

    -- set scan.timestamp-millis=1697018249001 for all tables
SET 'scan.timestamp-millis' = '1697018249001';
SELECT * FROM T;

-- set scan.timestamp-millis=1697018249000 for the table mycatalog.default.T
SET 'paimon.mycatalog.default.T.scan.timestamp-millis' = '1697018249000';
SELECT * FROM T;

-- set scan.timestamp-millis=1697018249000 for the table default.T in any catalog
SET 'paimon.*.default.T.scan.timestamp-millis' = '1697018249000';
SELECT * FROM T;

-- set scan.timestamp-millis=1697018249000 for the table mycatalog.default.T1
-- set scan.timestamp-millis=1697018249001 for others tables
SET 'paimon.mycatalog.default.T1.scan.timestamp-millis' = '1697018249000';
SET 'scan.timestamp-millis' = '1697018249001';
SELECT * FROM T1 JOIN T2 ON xxxx;

    Contributing #

    Apache Paimon 由一个开放且友好的社区开发。热忱欢迎每个人加入社区并为 Apache Paimon 做出贡献。参与社区和贡献 Paimon 的方式有很多，包括提问、提交缺陷报告、提出新功能建议、加入邮件列表讨论、贡献代码或文档、改进网站、测试发布候选版本以及撰写相关博客等。

    你想做什么？ #

    为 Apache Paimon 贡献不仅仅限于为项目编写代码。以下列出了帮助项目的不同途径：

    Area Further information
    Report Bug 要报告 Paimon 的问题，请前往 Paimon 的 Issues 页面。 请提供关于你遇到的问题的详细信息，如果可能的话，请添加有助于重现该问题的描述。
    Contribute Code 阅读 Code Contribution Guide
    Code Reviews 阅读 Code Review Guide
    Release Version 发布新的 Paimon 版本。
    Support Users 回复 用户邮件列表上的问题， 查看 Issues 中最新的问题，寻找实际用户提问的工单。
    宣传 Apache Paimon 组织或参加 Paimon 线下交流会，撰写 Paimon 博客，或者在 dev@paimon.apache.org 邮件列表 分享你的会议、交流会或博客文章。
    还有其他问题吗？请联系 dev@paimon.apache.org 邮件列表

    Code Contribution Guide #

    Apache Paimon 由志愿者的代码 contribution 维护、改进和扩展。我们欢迎大家为 Paimon 做出贡献。

    如有任何问题，欢迎随时提问。你可以发送邮件到开发者邮件列表，或在你参与的 issue 下发表评论。

    1Discuss

    创建一个 Issue 或者在邮件列表中发起讨论，达成社区共识。

    请求领取 issue 时，请注意这不仅仅是“请把这个 issue 分配给我”，你需要说明你对该 issue 的理解和你的设计方案，如果可能，还需要提供你的 POC（概念验证）代码。

    2Implement

    根据 Issue 中达成的共识创建 Pull Request。

    1. 只有在你被指派到该 Issue 后，才创建 PR。2. 请关联一个 Issue（如果有），例如 fix #123。 3. 请启用你自己的克隆项目的 Actions 功能。 .

    3Review

    与 reviewer 合作。


    1. 确保没有包含无关或不必要的格式化更改。2. 请确保测试通过。 3. 请不要自行关闭对话。

    4Merge

    Paimon 的 committer 会检查 contribution 是否符合要求，并将代码合并到代码库中。

    Code Review Guide #

    每次评审都需要检查以下六个方面。我们鼓励按照顺序检查这些方面，以避免在形式要求未满足或社区对变更无共识时，浪费时间在详细代码质量的评审上。

    1. contribution是否描述充分？ #

    检查contribution是否有足够清晰的描述以支持良好的评审。对于简单的变更和修复，不需要长篇描述。如果实现严格按照之前在 issue 或开发邮件列表上的讨论进行，只需简短地引用该讨论即可。

    如果实现与共识讨论中达成的方法不同，则需要对实现进行详细描述，才能继续对该contribution进行进一步评审。

    2. 某些 contribution 是否需要特定 committer 的关注和批准？ #

    某些变更需要特定 committer 的关注和批准。

    如果 pull request 需要特定人员关注，则应由被标记的 committer 或 contributor 中的某人给出最终批准。

    3. 整体代码质量是否良好，符合我们在 Paimon 中希望维持的标准？ #

    • 代码是否遵循正确的软件工程实践？代码是否正确、健壮、易维护、易测试？
    • 修改是否考虑了性能，特别是在性能敏感的部分？
    • 修改是否有足够的测试覆盖？测试执行是否快速？
    • 如果更改了依赖，NOTICE 文件是否已更新？

    代码规范可参考 Flink Java Code Style and Quality Guide.

    4. 文档是否更新？ #

    如果 Pull Request 引入了新特性，该特性应当有相应的文档说明。

    成为一名 Committer #

    当你做出足够多的 contribution 后，可以被提名为 Paimon 的 Committer。详见 Committer.

    概览 Overview #

    如果你定义了一个具有主键的表，则可以对该表执行插入、更新或删除操作。

    主键由一组列组成，这些列对于每条记录来说具有唯一值。Paimon 通过在每个 bucket 中对主键进行排序来强制数据有序，从而支持在主键上应用过滤条件以实现高性能查询。详见： CREATE TABLE。

    桶 Bucket #

    未分区表，或分区表中的各个分区，都会被进一步划分为 bucket，以为数据提供额外的结构，从而实现更高效的查询。

    每个 bucket 目录中包含一个 LSM 树，其内容包括： changelog files。

    每个 bucket 的范围是通过记录中一个或多个列的哈希值来确定的。
    用户可以通过设置 bucket-key 选项 来指定分桶列。
    如果未指定 bucket-key，则默认使用主键（如果定义了主键）或整条记录作为分桶键。

    Bucket 是读写的最小存储单元，因此 bucket 的数量会限制最大处理并行度。
    但这个数量也不宜过大，否则会产生大量小文件，导致读取性能下降。
    一般建议每个 bucket 中的数据量保持在约 200MB 到 1GB 之间。

    如果你希望在建表后调整 bucket 的数量，请参考 调整 bucket 数量。

    LSM Trees #

    Paimon 采用 LSM 树（Log-Structured Merge-Tree）作为文件存储的数据结构。本文档将简要介绍有关 LSM 树的基本概念。

    Sorted Runs #

    LSM 树将文件组织成多个 Sorted Run（有序运行）。每个 Sorted Run 由一个或多个数据文件组成，且每个数据文件只属于一个 Sorted Run。

    每个数据文件中的记录都按主键排序。在同一个 Sorted Run 内，各数据文件的主键范围不会发生重叠。

    如上所示，不同的 Sorted Run 之间可能存在主键范围的重叠，甚至可能包含相同的主键。当查询 LSM 树时，系统会合并所有 Sorted Run，并根据用户指定的 Merge Engine 以及每条记录的时间戳，对具有相同主键的记录进行合并。

    新写入 LSM 树的记录会先缓存在内存中。当内存缓冲区满时，所有记录会被排序后刷新到磁盘上，从而创建一个新的 Sorted Run。
